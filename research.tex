\subsection{Поставновка эксперимента}

Исходные данные для экспериментов.

Для проведения экспериментов определена область допустимых значений изменяемых параметров:

\begin{itemize}

\item рассматриваемые сети 802.11s, LTE;

\item количество узлов сети 4-16;

\item фрагмент проезжей части 800 м. с 4-я поворотами 90 градусов и с двусторонним движением;

\item интенсивность передачи данных от узла к облачной среде 8-2048 Кбит/сек;

\item размер сообщений 1024 байт;

\item протоколы маршрутизации HWMP, OLSR, AODV, DSDV;

\item тип трафика TCP, UDP;

\item до четверти узлов mesh сети оснащены интерфейсом LTE (осуществляет передачу данных в облачную среду), данные от остальных участников сети передаются через них.

\end{itemize}

Заданная такими границами область значений параметров моделирования позволяет исследовать наиболее динамичные периоды существования mesh сети (короткое время жизни сети, широкий диапазон интенсивности сетевого трафика, высокая интенсивность перераспределения маршрутов).

\subsection{Анализ результатов исследования}

\subsubsection{Оценка отношений расчётной и фактической скоростей передачи сообщений.} 
Одной из основных причин появления отличий между реальной и рассчётной скоростями передачи данных является использование служебного трафика, в частности, необходимого для обеспечения работы протоколов маршрутизации, что особенно актуально в мобильных mesh сетях для поддержки актуальности данных о состоянии топологии сети. Исследование фактической скорости передачи данных осуществлялось для разных протоколов маршрутизации и малом времени существования сети. Для сравнения были рассмотрены следующие протоколы маршрутизации OLSR, DSDV, AODV, HWMP. В эксперименте использовался UPD трафик передаваемый со скоростями 8, 32, 64, 128,512, 1024, 2048 Кбит/сек. \subsubsection{Оценка достоверности доставки сообщений}
 
Достоверность доставки сообщений оценивалась для наиболее сложных случаев существования сети, в которых изменение структуры сети происходит в течении 1 сек. Для данной ситуации характерна постоянная работа протоколов маршрутизации и процент потерь сообщений может быть значительным. Моделирование проводилось со следующим набором параметров:

\begin{itemize}

\item Число подвижных узлов-ретрансляторов фиксировано --- 16.

\item Скорость передачи сообщений (от 8 до 2048 Кбит/сек.).

\item Размер передаваемого сообщения фиксирован --- 1024 байт.

\item Максимальная корость движениея узла 50 км/час.

\item Фрагмент проезжей части 800м с 4-я поворотами 90 градусов.

\item 4 протокола маршрутизации: OLSR, HWMP, DSDV, AODV.

\end{itemize}

Наибольшие потери сообщений возникают при использовании протокола OLSR, который не следует использовать для передачи потоковых данных без применения усложнённых механизмов передачи.

\subsubsection{Оценка среднего значения задержки отклика от облачной среды}

Исследование среднего времени передачи сообщения от абонента к облаку определялась для сетей с 8 и 16 мобильными узлами, при использовании протокола маршрутизации HWMP.

В процессе моделирования были заданы следующие значения параметров:

\begin{itemize}

\item средняя скорость передачи 512 Кбит/сек,

\item TCP трафик,

\item размер сообщения 1024 байт,

\item фрагмент четырёхполосной проезжей части 800 м. с 4 поворотами 90 градусов,

\item максимальная скорость движения узлов 50 км/ч.

\end{itemize}

Для 8 узлов (1 автомобиль на 100 м. проезжей части), при этих параметрах, среднее время доставки сообщения от абонента в облачную среду составило 70,41 мс.; для 16 узлов 4,04 мс. Уменьшение времени доставки сообщений в сети с 16-ю узлами (2 автомобиля на 100 м. проезжей части) связано возможностью построения дополнительных маршрутов передачи данных.

В рамках данного эксперемента проводились исследования возможности доставки экстренного сообщения от аварийного автомобиля до автомобиля, оснащённого интерфейсом LTE для передачи экстренного сообщения в облачную среду. Передача сообщения от аварийного автомобиля возможна при условии соединения аварийного автомобиля с мобильной mesh-сетью средствами 802.11s. В первом эксперименте определялся процент потерь сообщений от максимальной скорости движения автомобилей при разной плотности потока автомобилей оснащённых интерфейсом 802.11s.(рис.9).

Процент потерь сотавляет 78-92\%. Из этого следует, что при таком режиме движения необходимо либо увеличивать плотность машин находящихся в радиусе действия передатчика mesh (802.11s), что позволит использовать больше альтернативных путей доставки сообщений. Другим вариантом решения проблемы с небольшой плотностью машин, оснащенных передатчиками 802.11s, является использование альтернативной технологии передачи - DSRC или использование прямых глобальных каналов точка-точка (LTE, WiMAX).

Во втором эксперименте проводилось исследование возможности доставки сообщения от аварийного автомобиля, средствами 802.11s до хотя бы одного автомобиля оборудованного 802.11s и LTE на прямом участке 4-х полосного шоссе протяжённостью 800 метров (рис.10).

В этом случае целесообразно использовать альтернативные протоколы отправки сообщений в облачную среду, такие как DSRC (при наличии покрытия дороги).
